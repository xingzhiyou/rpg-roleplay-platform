# 存档管理

## 这页能干嘛

「存档管理」是查看、创建、切换游戏存档的地方。每个存档绑定一本剧本和一张角色卡，记录全部游戏历史。每完成一回合对话，系统自动生成一个新的分支节点（类似 git commit）。

入口：顶栏「**开始游戏**」→「存档目录」。

![存档目录主界面](./screenshots/saves-list.png)

---

## 关键概念

### 列表与详情

存档列表显示列：**存档**名称、**剧本**、**玩家**、**节点**数、**最后游玩**时间、**状态**（在玩 / 未激活）。

点击某条存档，右侧（或下方）展开详情面板，包含三个标签：

- **概览**：剧本、玩家、回合数、状态、分支节点数、故事时间、最后游玩、创建时间，以及最新对话片段
- **设置**：游戏设置（元知识 / 引导 / 防剧透），部分字段建档后锁定
- **分支**：分支节点列表及「打开完整分支树」入口

### 当前存档（在玩）

存档列表里标记「在玩」的是当前激活存档，进入游戏默认加载这条。点「**设为当前**」可切换。

### 分支

存档历史记录像 git 一样组织为有向树。从末尾节点继续 → 沿原分支；从中段历史节点继续 → 自动新建分叉，原分支保留。

## 常见任务

### 开始新游戏

点右上角「**开始新游戏**」进入向导，填写以下内容后点「**创建并进入**」：

1. **存档名称**（必填）+ **剧本**（必填）
2. **选择角色**：使用现有角色卡，或新建一张
3. **出生点**：选择故事开局的时间锚点（按剧情阶段划分）
4. **角色初始身份卡**（可选）：叠加在角色卡之上的身份 overlay（穿越者 / 卧底等），可让 AI 生成推荐或手动创建
5. **剧情期望**（可选）：告诉 GM 你希望的剧情走向

![新游戏向导](./screenshots/saves-new-game-wizard.png)

> 如果没有导入任何剧本，需要先去「剧本管理」导入剧本后才能创建存档。

### 继续游戏

- 点顶栏右上角「**进入当前游戏**」，直接进入当前存档的最新节点。
- 或在列表选中存档 → 详情面板「**继续游戏**」按钮。
- 或在列表点「继续」快捷按钮。

### 切换到某个历史节点（回溯 / 分叉）

1. 选中存档 → 详情面板「**分支**」标签，查看节点列表。
2. 点「**打开完整分支树**」进入分支图全页。
3. 点击目标历史节点 →「激活」。
4. 再进入游戏，从该节点继续。若选中节点不是当前分支末端，进入后会自动新建分叉。

### 导出存档

选中存档 → 详情面板「**导出**」→ 弹出导出面板，选择档位后下载。

导出的是一个 **zip 自包含包**，内含剧本 + 章节 + 知识库（可选向量）+ 存档进度，可导入到**任何实例（含开源自托管）**，不依赖本服务器。

**导出档位**（弹窗实时显示各档大小供参考）：

| 档位 | 内容 | 适用场景 |
|---|---|---|
| **完整** | 含向量索引 | 导入即用，RAG 检索立即可用；包体较大 |
| **标准**（默认） | 去除向量 | 包体更小，导入端重建嵌入后可检索；推荐日常备份 |

> **注意**：只能导出**自己拥有的剧本**对应的存档；订阅的公开剧本因版权限制不可打包导出。

### 导入存档

右上角「**导入存档**」→ 上传 `.zip` 文件（旧格式 `.json` 也仍支持）→ 系统自动识别格式，导入后在列表新增一条记录（不自动激活）。

若导入的是**标准档**（去向量），需到「剧本管理」页面找到对应剧本，点「**嵌入**」重建向量检索后 RAG 才能正常工作。

### 重命名 / 删除存档

选中存档 → 详情面板操作菜单 →「**重命名**」或「**删除**」。删除不可撤销（服务器 commit 文件在保留期内可由管理员恢复）。

## 常见问题

**存档列表是空的？**
点「**开始新游戏**」创建第一条存档。需要先在「剧本管理」导入至少一本剧本。

**进游戏后剧情从错误节点开始？**
退出 → 打开「分支图」→ 找到正确节点 →「激活」→ 再次进入游戏。

**分支图怎么看？**
每列（swimlane）是一条分支轨道，圆圈是节点，空心圆圈代表当前 HEAD，虚线代表已删除的子树。详见[分支图](branches.md)。

**存档设置里哪些字段不能改？**
「建档锁死项」（如起始世界线、初始身份）在游戏开始后只读，改动请求会被后端拒绝并提示。
