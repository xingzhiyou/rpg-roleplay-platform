# MCP 工具

## 这页能干嘛

「MCP」（Model Context Protocol）页管理你配置的 MCP 服务器。MCP 是一种让 GM 调用外部工具的标准协议，比如让 AI 能读取本地文件、调用 API、执行搜索等扩展能力。

入口：顶栏「全部功能」→「MCP」（路由：`#mcp`）。

![MCP 工具界面](./screenshots/mcp-main.png)

---

## 关键概念

### 什么是 MCP 服务器

MCP 服务器是一个独立进程，实现了 MCP 协议，暴露若干「工具（tool）」供 GM 调用。当前平台支持：

- **本地 MCP**：在你自己机器上运行的 MCP 进程（需要本地安装）
- **服务器侧 MCP**：托管在服务器上的 MCP 服务

### 与插件的区别

MCP 侧重于工具调用（函数执行），插件侧重于平台功能扩展。MCP 工具在每轮 GM 调用时可被主动使用。

## 常见任务

### 添加 MCP 服务器

1. 进入 MCP 页，点「**添加**」（或按页面引导操作）。
2. 填写服务器名称、连接方式（本地 / 服务器）和必要的连接参数。
3. 保存后该 MCP 服务器的工具列表会显示在页面上。

### 启用 / 禁用 MCP 工具

找到对应工具，切换启用开关。禁用后 GM 不再使用该工具。

---

## 常见问题

**GM 没有使用 MCP 工具？**
确认：①工具已启用；②当前使用的模型支持 tool use（工具调用）能力；③MCP 服务器进程正在运行且可达。
