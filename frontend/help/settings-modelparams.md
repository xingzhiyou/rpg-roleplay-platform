# 模型参数调优

## 这页能干嘛

调整发送给 AI 模型的采样参数，控制 GM 输出的随机程度、长度、重复抑制等。这些参数影响所有 API 调用，是「模型说话方式」的旋钮，不是内容审查，而是文风和风格的精调。

---

## 关键概念

- **Temperature（温度）**：控制输出随机性。越高越天马行空，越低越保守确定。范围 0–2，默认 0.78。
- **Top-p**：累积概率截断。模型从累积概率达到 top-p 的那些词中采样，较低的值会让输出更集中。常用 0.90–0.95。
- **Top-k**：只从概率最高的 K 个词中采样。0 表示关闭。
- **Max Tokens**：单轮回复的最大输出长度。过短会导致 GM 叙事被截断。
- **上下文窗口（Context Size）**：每次请求携带的历史和记忆上限。超过后自动截断旧内容。
- **重复惩罚（Repetition Penalty）**：抑制模型反复用同样的词；1.0 = 无效果，1.15–1.2 是常用范围。
- **频率惩罚（Frequency Penalty）**：OpenAI 系参数，根据词在已生成文本中出现的频率来降低概率。
- **存在惩罚（Presence Penalty）**：OpenAI 系参数，根据词是否已经出现过来降低概率（与频率无关）。
- **Reasoning Effort**：仅对推理型模型（如 o3、DeepSeek R1）显示。控制模型「想多深」：low = 快速省 token，high = 最深推理。
- **种子（Seed）**：-1 表示每次随机，固定数字则让相同输入产生相同输出（可复现）。
- **停止序列（Stop）**：遇到这些字符串时立刻停止生成，多条用 `|` 分隔。

## 不同场景的参数建议

### 创意叙事（开放沙盒、诗意风格）

适合调高 temperature（0.9–1.1），稍提高 top_p（0.95–0.98），可以适当降低 repetition_penalty（1.05 左右）让模型不过分回避熟悉的意象。

### 规则裁定 / 系统指令解析

需要准确，调低 temperature（0.2–0.4），降低 top_p（0.8–0.88），重复惩罚默认即可（1.05–1.1）。这是「保守」预设的思路。

### 对话驱动 RPG（角色扮演为主）

平衡预设通常已够用。如果觉得 GM 说话太单一，可以微调 presence_penalty（+0.1 左右）来鼓励用新词。

### 长篇叙事 / 史诗战争场面

增大 max_tokens（2048–4096），确保叙事不被截断。上下文窗口选 32K 以上保留足够历史。

### 推理型模型（o3、DeepSeek R1）

这类模型有「thinking」步骤，temperature 对它们影响较小。主要调节：
- **Reasoning Effort**（仅在选中推理型模型时显示）：low 速度快，high 思考最深

## 参数差异：不同供应商

| 参数 | OpenAI / OpenAI 兼容 | Anthropic | 推理型（o3 / R1 等） |
|---|---|---|---|
| temperature | 支持 | 支持 | 部分支持（有上限） |
| top_p | 支持 | 支持 | 部分支持 |
| top_k | 不支持 | 支持 | — |
| frequency_penalty | 支持 | 不支持 | — |
| presence_penalty | 支持 | 不支持 | — |
| repetition_penalty | 部分支持 | 不支持 | — |
| reasoning_effort | 不支持 | 不支持 | 支持（o3 系列） |
| thinking_budget | 不支持 | 支持（Claude 3.7+） | — |
| mirostat | 本地 vLLM 等 | 不支持 | — |

> 不支持的参数会被后端自动过滤，不会导致 API 报错，但也不会生效。如果某个参数看起来没效果，先确认当前模型是否支持它。

## 常见任务

### 任务 1：切换到预设并微调

步骤 1. 点击顶部预设按钮（如「平衡」），参数栏自动更新。

步骤 2. 拖动下方任意滑块进行微调，预设自动切换为「自定义」。

步骤 3. 参数会即时保存，无需手动确认。

![模型参数滑块界面](./screenshots/settings-modelparams-sliders.png)

### 任务 2：查看实际发送的参数

页面底部「预览 JSON」区域实时显示当前参数组合发送给 API 的 JSON 格式，方便核对是否符合预期。

### 任务 3：配置 NSFW / 成人内容策略

页面中部有「NSFW · 成人内容」控制：

- **禁止**：GM 不生成任何相关描写
- **含蓄**：允许暗示，避免直接描写
- **开放**：允许明确描写，由 GM 主动推进程度可通过「NSFW 强度」控制
- **露骨**：完全开放，GM 可主动推进

所有模式下，对未成年角色的性化描写都会被系统拦截。

「NSFW 额外约束」文本框可以附加自然语言限制，写入系统提示词中。

## 常见问题

**Q: 调高 temperature 之后 GM 的输出变得语无伦次怎么办？**
A: 先降回 1.0 以内，同时适当降低 top_p（0.90 左右）。temperature 超过 1.2 在很多模型上容易出现质量下降。

**Q: max_tokens 设多少合适？**
A: 一段正常 GM 叙事大约 200–600 字，对应 300–900 token。设置 1024 是安全起点；如果你的剧本需要长段落，可以设到 2048–4096。注意 max_tokens 增大也会增加 API 费用。

**Q: Reasoning Effort 选项我的模型没有显示？**
A: 只有当前选中的模型具有「推理」能力标签时才会显示。如果你用的是 o3、DeepSeek R1 等推理型模型但看不到这个选项，检查模型能力标签是否正确配置（见[API 设置](./settings-models.md)）。

**Q: 停止序列填了 `玩家:` 有什么效果？**
A: GM 输出遇到文本 `玩家:` 时立刻停止。这通常用于防止 GM 替玩家说话（越界扮演玩家角色）。
