# 角色卡

## 这页能干嘛

「角色卡」页管理你的**用户角色卡**（PC 卡）——即你在游戏里扮演的角色。这些卡跨剧本、跨存档共享，绑定到你的账户，不属于某本具体剧本。

入口：顶栏「角色卡」→「用户角色卡」。

![角色卡主界面](./screenshots/cards-list.png)

---

## 关键概念

### 用户角色卡 vs NPC 角色卡

| | 用户角色卡 | NPC 角色卡 |
|---|---|---|
| 代表谁 | 你扮演的 PC | 剧本里的配角（LLM 提取或手动创建） |
| 归属 | 账户级，跨剧本共享 | 绑定到某本剧本 |
| 入口 | 本页（角色卡） | 剧本管理 → 详情 → NPC 角色卡 |
| 可见范围 | 私有 / 公开 | 剧本内 / 私有 / 公开 |

### 角色卡字段说明

编辑表单包含以下字段（展开区块操作）：

**基本信息**
- **姓名**（必填）：GM 识别角色的主名
- **全名（full name）**：欧美式全名，与姓名不同时填
- **身份 / 职位**：社会身份，如「南陵巡检」
- **别名**：逗号分隔，GM 据此识别同一角色的多种称呼
- **标签**：逗号分隔，用于搜索和筛选

**人物画像**
- **前史 / 背景**：登场前的来历
- **外貌**
- **性格 / 设定**
- **语气 / 说话风格**：GM 模仿口吻的参考
- **当前状态**：开局时的处境或心境

**剧情与对话**
- **关键秘密**：仅 GM 可见，不会直接暴露
- **示例对话**：每行一句，帮助 GM 模仿语气

**注入参数**：控制该卡在 GM 上下文里的行为
- **重要度**（0–100）：越高越优先注入，主角建议 80+
- **Token 预算**：注入时占用的最大 token 数，默认 450
- **优先级**：同等重要度时的排序，默认 100
- **可见范围**：私有 / 公开
- **启用**：关闭后该卡不参与注入

### 可见范围

- **私有**：仅自己可见（默认）
- **公开**：所有用户可浏览

## 常见任务

### 新建用户角色卡

右上角「**新增角色卡**」→ 填写姓名（必填）→ 填写其他字段 →「**创建**」。

### 导入 SillyTavern 角色卡（酒馆卡）

支持 `.png`（PNG 内嵌元数据）、`.json` 和 `.webp` 格式，单次最多 8 个文件。

1. 右上角「**导入酒馆卡**」。
2. 把文件拖入虚线框，或点击选择文件。
3. 系统自动解析并显示预览（名称、格式、简介、开场白、标签）。
4. 确认后点「**导入**」。

也可以在导入弹窗切到「粘贴 JSON」，直接粘贴角色卡 JSON 文本后解析导入。

![导入酒馆卡弹窗](./screenshots/cards-import-tavern.png)

### 编辑角色卡

点击列表中某行角色卡 → 下方详情面板展开 →「**角色设置**」标签 → 修改字段 →「**保存**」。

### 搜索和筛选

搜索栏支持按名称 / 身份 / 标签搜索；右侧下拉可按「全部 / 私有 / 公开」筛选。

### 跨剧本复用

用户角色卡账户级共享。开始新游戏时，向导的「选择角色」步骤里会列出你所有的用户角色卡。

## 常见问题

**角色卡的 Token 预算设多少合适？**
主角建议 600–800，重要 NPC 400–600，次要 NPC 200–400。总注入量不超过模型上下文窗口的 1/4 为宜。

**导入 PNG 卡看不到内容？**
部分 PNG 卡的元数据格式不标准。可以尝试「粘贴 JSON」方式导入对应的 JSON 文件。

**NPC 角色卡在哪里管理？**
NPC 角色卡属于剧本，入口在「剧本管理 → 选中剧本 → NPC 角色卡」标签，不在本页。
