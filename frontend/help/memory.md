# 记忆设置

## 这页能干嘛

「记忆」设置页控制 GM 的记忆召回行为——每轮游戏时从历史对话中检索多少内容、如何压缩历史、各记忆桶（Bucket）是否启用。调整这里的参数会影响 GM 对历史信息的引用深度和准确度。

入口：顶栏「设置」→「记忆」（路由：`#settings-memory`）。

![记忆设置界面](./screenshots/settings-memory.png)

---

## 关键概念

### 记忆桶（Bucket）

记忆被分成三类桶，各自独立检索：

- **固定记忆桶**：玩家或系统手动钉住的重要信息（如关键剧情转折、重要 NPC 关系），每轮都会优先注入
- **世界记忆桶**：世界观类记忆（地名、势力、规则等）
- **角色记忆桶**：角色关系类记忆（NPC 态度、关系状态等）

时序桶（最近历史摘要）始终启用，不可关闭。

---

## 设置项说明

### 记忆 · 召回行为

| 设置 | 说明 | 范围 |
|---|---|---|
| **默认召回深度** | 每轮从原文检索的最大段数 | 2–20 |
| **历史摘要窗口** | 最近 N 个回合压缩为摘要喂入 | 3–20 |
| **每轮注入记忆 token 上限** | 每次请求注入到上下文的记忆内容最大 token | 200–2000 |
| **N 轮后自动归档老记忆** | 超过此回合数的记忆条目自动归档到事实库 | 10–200 |

### 记忆 · 记忆桶配置

| 设置 | 说明 |
|---|---|
| **固定记忆桶上限** | 固定桶最多保存的条目数（5–100），超出后旧条目转入事实库 |
| **启用固定记忆桶** | 关闭后不再维护固定桶，召回仅走时序桶和角色桶 |
| **启用世界记忆桶** | 关闭后跳过世界观类记忆的检索注入 |
| **启用角色记忆桶** | 关闭后跳过角色关系类记忆的检索注入 |

## 常见任务

### 提高 GM 的历史记忆深度

增大「默认召回深度」（建议 8–12 对长篇剧本），同时适当增大「每轮注入记忆 token 上限」（建议 800–1200）。注意上下文总 token 预算有限，召回越深成本越高。

### 关闭某类记忆桶

找到对应的「启用 XXX 桶」开关，切到「关闭」，即时保存。可在游戏中随时切回开启。

### 查看游戏中实际召回了什么

进入游戏 → 右侧面板「上下文」标签 → 可查看当轮实际注入给 GM 的记忆内容。

---

## 常见问题

**GM 总是忘记重要事件，怎么解决？**
1. 增大「默认召回深度」
2. 进入游戏后，在右侧「记忆」面板手动添加固定记忆，钉住关键信息
3. 确认「启用固定记忆桶」为开启状态

**记忆桶设置修改后什么时候生效？**
下次 GM 调用时立即生效，不需要重启游戏。
