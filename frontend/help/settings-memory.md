# 记忆配置

## 这页能干嘛

控制游戏的记忆系统如何工作：GM 能「记住」多少历史、哪些内容被检索进当前对话、旧记忆什么时候自动归档。调整这里的参数可以平衡 GM 的记忆广度、上下文费用和响应速度。

---

## 关键概念

### 记忆桶

系统把记忆按类型分成三个桶，分别存放不同性质的内容：

- **固定记忆桶（Pinned）**：你或 GM 手动钉住的重要事实，优先级最高，每轮必然注入。例如：「主角失去了左手」「当前城市是废都边境」。
- **世界记忆桶（World）**：与世界观、地点、派系、规则相关的背景信息。GM 会按相关性检索部分注入。
- **角色记忆桶（Character）**：人物关系、角色性格、NPC 状态等。按相关性检索注入。

三个桶都可以单独开启或关闭（在「记忆桶配置」区域控制）。

### 召回 vs 注入

- **召回**：每轮对话时，系统从记忆库中检索出与当前剧情最相关的片段。
- **注入**：把召回的内容塞进本轮的 AI 提示词里，让 GM 「看到」。注入内容过多会消耗大量 token。

---

## 参数说明

### 召回行为

| 参数 | 含义 | 默认值 | 建议范围 |
|---|---|---|---|
| 默认召回深度 | 每轮从原文检索的最大段数 | 6 | 2–20，记忆多时调大 |
| 历史摘要窗口 | 最近 N 个回合压缩成摘要再喂入 | 8 | 3–20 |
| 每轮注入记忆 token 上限 | 注入到上下文的记忆内容最大 token 数 | 800 | 200–2000 |
| N 轮后自动归档老记忆 | 超过此回合数的记忆条目自动归档到事实库 | 50 | 10–200 |

### 记忆桶配置

| 参数 | 含义 | 默认 |
|---|---|---|
| 固定记忆桶上限 | 固定记忆桶最多保存的条目数，超出后旧条目转入事实库 | 20 |
| 启用固定记忆桶 | 关闭后召回只走时序桶和角色桶 | 开启 |
| 启用世界记忆桶 | 关闭后跳过世界观类记忆的检索注入 | 开启 |
| 启用角色记忆桶 | 关闭后跳过角色关系类记忆的检索注入 | 开启 |

---

## 自动写入 vs 手动钉

- **自动写入**：GM 每轮叙事结束后，Extractor 会自动把重要状态变化写入记忆库（如角色受伤、获得道具、关系变化等）。这是默认行为，无需干预。
- **手动钉（Pin）**：在游戏内侧栏可以把某条记忆标记为「固定」，钉住的内容进入固定记忆桶，每轮必然出现在上下文中，不会被自动归档删除。适合你认为极其重要、GM 绝对不能忘的事实。

> 固定记忆桶有上限（默认 20 条）。超出后最旧的固定记忆会转移到事实库，不再每轮强制注入，但仍可被召回。

## 参数调整建议

### 剧情早期（少量记忆）

使用默认设置即可。召回深度 6、注入上限 800 token 对大多数对话都够用。

### 剧情中后期（大量历史记忆）

- 提高**召回深度**（8–12）：让 GM 能看到更多相关历史片段。
- 提高**每轮注入 token 上限**（1000–1500）：容纳更多召回内容。
- 适当降低**自动归档轮数**（30–40）：加快旧记忆归档，避免记忆库膨胀影响性能。

### 注重节省 API 费用

- 降低**每轮注入 token 上限**（400–600）：最直接减少每轮消耗。
- 关闭不需要的记忆桶：如果剧本没有复杂世界观，可以关闭「世界记忆桶」。

### 大量 NPC 的群戏剧本

- 确保「角色记忆桶」开启。
- 提高召回深度（10–15）让更多角色关系被检索到。

## 常见任务

### 任务 1：调整召回深度

步骤 1. 进入「平台设置 → 记忆」，找到「默认召回深度」滑块。

步骤 2. 拖动滑块或在右侧数字框直接输入数值（范围 2–20）。

步骤 3. 松开滑块（或输入框失去焦点）后自动保存。

![记忆召回行为设置](./screenshots/settings-memory-recall.png)

### 任务 2：关闭某个记忆桶

步骤 1. 进入「记忆 · 记忆桶配置」区域。

步骤 2. 找到「启用世界记忆桶」或「启用角色记忆桶」的开关，点击切换为「关闭」。

步骤 3. 立即生效，之后的对话不再检索该桶的内容。

### 任务 3：清理固定记忆（防止桶满）

固定记忆桶满后旧条目会自动转移到事实库。如果你想主动管理，在游戏内侧栏「记忆」面板中可以取消钉住某条记忆，或直接删除。

## 常见问题

**Q: GM 总是忘记某个重要设定，怎么办？**
A: 在游戏内侧栏把该设定手动钉住（Pin），进入固定记忆桶后每轮必然注入，GM 不会忘记。

**Q: 召回深度调太大有什么副作用？**
A: 每轮注入的 token 数量增加，API 费用上升，且可能超过「每轮注入 token 上限」而被截断。建议先调上限再调深度。

**Q: 记忆归档之后内容消失了吗？**
A: 没有，归档只是从「每轮必然注入」变成「按相关性检索才注入」，事实库里的内容还在，相关时仍会被召回。

**Q: 「历史摘要窗口」调大有什么效果？**
A: 更多回合的对话历史会被压缩成摘要，帮助 GM 保留更长的故事脉络。但压缩摘要本身会消耗一次额外的模型调用，适度设置即可。

**Q: token 上限和上下文窗口有什么关系？**
A: 「每轮注入记忆 token 上限」只控制记忆部分的大小，整体上下文窗口（含历史对话、系统提示等）在「模型参数」页的「上下文窗口」里配置。记忆上限不能超过上下文窗口的总额度。
