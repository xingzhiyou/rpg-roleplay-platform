# 输入框 / 命令 / 附件

游戏主界面底部的输入区是你与 GM（游戏主持人）对话的核心入口，除了普通对话，它还支持命令、附件、角色提及和权限控制。

![游戏输入框全貌](./screenshots/game-composer-overview.png)

---

## 普通对话

在输入框里直接写你想做的事、说的话、或给出行动描述，然后按 **回车**（或 **⌘ + 回车**）发送。GM 会读取你的输入，结合当前剧情状态给出叙事回应。

- **回车** = 发送；**Shift + 回车** = 在输入框内换行
- 输入框支持自动撑高，最多展开到 280px 高度
- 发送后 GM 流式输出，可随时点「停止」中断

如果当前场景下玩家角色处于被动状态（昏迷、旁观、过场），点击工具栏里的「**继续**」按钮可直接让 GM 推进一段剧情，无需输入任何文字。

---

## / 命令

在输入框里输入 `/` 会弹出命令选择菜单，用 **↑↓** 选择、**回车** 确认、**Esc** 取消。也可以直接打出命令全名后跟参数。

![命令菜单弹出效果](./screenshots/game-composer-command-menu.png)

### 查询类

| 命令 | 说明 |
|---|---|
| `/status` | 显示当前状态摘要（角色、位置、时间等） |
| `/debug` | 查看上一轮 GM 的内部检索结果，帮助排查问题 |

### 状态写入类

| 命令 | 示例 |
|---|---|
| `/set 自然语言描述` | `/set 时间=次日清晨；地点=雾港码头` — 用自然语言批量强制改一组游戏参数，写入即落盘，优先级高于 GM 自动派生 |
| `/loc 地点` | `/loc 雾港码头` — 快速更新所在位置 |
| `/time 时刻` | `/time 次日清晨` — 推进时间线 |
| `/rel 角色 关系` | `/rel 沈知微 同盟` — 更新人物关系 |
| `/var 变量=值` | `/var phase=港口黄昏` — 设置世界线自定义变量 |

### 记忆类

| 命令 | 说明 |
|---|---|
| `/pin 文本` | 把一段内容加入固定记忆，GM 每轮都会参考 |
| `/note 文本` | 玩家私人笔记，不影响 GM |

### 模式类

| 命令 | 说明 |
|---|---|
| `/memory normal\|deep\|off` | 切换记忆检索模式（normal = 默认；deep = 精细检索；off = 关闭） |
| `/permission default\|review\|read_only\|full_access` | 切换 GM 写入权限（见下方「权限控制」） |

### 工程类

| 命令 | 说明 |
|---|---|
| `/save` | 手动保存当前存档 |
| `/retry` | 重试 GM 上一轮输出（换个结果） |

## 逐轮重新生成

每条 GM 消息旁（分支按钮边上）有「**重新生成这一轮**」按钮，点击后：

1. 存档回滚到该轮**之前**的状态
2. 用**同样的玩家输入**重走完整 GM 流程
3. 生成完毕，你得到一个结果不同的新回复

**与「从这里开始新分支」的区别**：

| 操作 | 玩家输入 | 分支行为 |
|---|---|---|
| **重新生成这一轮** | 不变，原样复用 | 替换当前轮，不新增分支节点 |
| **从这里开始新分支** | 可以修改 | 岔出一条新分支轨道，原分支保留 |

如果你只是不喜欢这一轮的叙事结果、想换个写法，用「重新生成」；如果想改变选择或走另一条路，用「开始新分支」。详见[分支图](branches.md)。

## @ 提及角色

在输入框里输入 `@` 会弹出角色列表，列出当前剧本的角色卡。选择后会在输入框里插入 `@角色名`，GM 会把这个标注视为你在针对该角色发言或行动。

## 附件（+ 按钮）

点击工具栏左侧的 **+** 图标打开附件菜单，可以附加以下内容：

![附件菜单](./screenshots/game-composer-attach-menu.png)

### 本地文件

- **文件 / 文本**（TXT · MD · JSON）：上传一段文字或文档，GM 会在本轮参考其中内容
- **图片**（PNG · JPG · WEBP）：上传图片，GM 支持多模态理解，可以描述、分析图中内容

### 剧本资源

- **插入章节**：从当前剧本目录搜索一个章节，注入为上下文
- **角色卡**：搜索或拖入一张角色卡，让 GM 参考该角色的详细设定
- **世界书条目**：全文搜索世界书，把匹配条目注入 GM 的上下文

### 能力扩展

- **MCP 工具**：挂载一个已配置的 MCP 服务器工具，让 GM 在本次对话里调用外部能力（详见「MCP 是什么」）
- **Skill 包**：挂载一个规则裁定包（如骰子规则 / 5E 规则集），让 GM 按照规则包进行裁定
- **计划模式**：预演一条叙事路径，但不写回存档状态

## 权限控制

工具栏下方显示当前 **LLM 写入权限**，点击可切换：

| 模式 | 说明 |
|---|---|
| **只读 · 纯叙事** | GM 所有状态写入都需要玩家手动审批，最保守 |
| **默认权限** | 白名单字段（位置、时间等）自动写入，其他需审批 |
| **自动审查** | 白名单 + 关系 / 世界变量也自动写入，其他需审批 |
| **完全访问** | 除硬黑名单外全部自动写入（类似 full-auto 模式） |

当 GM 提出写入申请或询问时，输入框上方会出现**待确认条目**，你可以逐条允许或拒绝。

---

## 模型切换

工具栏右侧显示当前 GM 使用的 AI 模型，点击可切换。模型旁的彩色圆点表示健康状态：绿色可用、橙色降级、红色不可达。切换只影响当前存档，不影响其他存档。

---

## 上下文用量

工具栏右侧有一个圆环进度指示，显示当前上下文占用比例。点击可查看详细的上下文分布（记忆 / 世界书 / 对话历史等各部分的 Token 占用）。进度超过 70% 变橙色，超过 90% 变红色，此时建议存档后开启新对话。
