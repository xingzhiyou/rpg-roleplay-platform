# 分支图

## 这页能干嘛

「分支图」用可视化的方式展示一个存档的全部分支历史，每个圆圈是一次游戏节点（类似 git commit），每列（swimlane）是一条分支轨道。你可以在这里查看所有分叉、切换到历史节点、删除不想要的分支子树。

入口：存档管理 → 选中存档 → 详情面板「**分支**」标签 →「**打开完整分支树**」，或顶栏「开始游戏」→「分支」。

![分支图界面](./screenshots/branches-graph.png)

---

## 关键概念

### 图例

- **列（swimlane）**：一条分支轨道，从左到右越靠右是越晚创建的分叉
- **实心圆圈**：一个普通节点（commit）
- **空心圆圈**（圆环）：当前 HEAD / active 节点，即当前游戏从哪里继续
- **虚线**：已删除的子树（保留显示以便溯源）

## 常见任务

### 切换到某个历史节点

1. 在分支图中找到目标节点，点击它。
2. 弹出节点详情，点「**激活**」。
3. 系统将 runtime 切换到该节点状态，再进入游戏时从该节点继续。

> 若选中的节点不是当前分支末端，进入游戏后会自动新建分叉，原分支保留。

### 从分支图直接进入游戏

右上角「**进入当前分支**」，或点击当前 HEAD 节点 →「继续」。

### 删除某个分支子树

1. 点击分支图中你想删除的起点节点。
2. 在弹出的操作区选「**删除此节点及其子树**」。
3. 输入「删除整棵子树」确认文字 → 确认删除。

> 删除在本存档中不可恢复。服务器端 commit 文件在保留期内可由管理员恢复。

### 刷新分支图

右上角「**刷新**」按钮，重新从服务器加载最新状态。

## 常见问题

**分支图是空的？**
说明当前存档还没有任何游戏记录。进入游戏完成至少一回合后，节点才会生成。

**怎么找到自己想回到的历史节点？**
可以看节点编号（从 1 开始），或者通过节点旁显示的时间戳（如果有）定位。

**分叉后原来的分支还在吗？**
在。原分支完整保留，新内容出现在新分叉的轨道上。两条线都显示在分支图中。
