# P — Stellatrix 桌面端 UI 设计说明（控制台 + 应用窗口）

> 给设计师的完整功能说明。目标:据此在 Claude Design 设计桌面端 UI。
> 桌面端 = Electron 应用,**渠道 B**(开源仓 GitHub Releases 分发,非 App Store)。
> 代码脚手架见 `desktop/`;本文是「该长什么样、有哪些屏/控件/状态/流程」的规格。

---

## 0. 一句话定位

一个**本地服务端控制台**桌面应用。它不是游戏本身,而是「管理游戏后端服务」的运维面板 +
一个加载游戏 Web 界面的窗口。一个 app,两种模式:

- **在线模式**:窗口直接连云端 `play.stellatrix.icu`,即开即用,数据在云端账号。
- **本地模式**:在本机启动捆绑的数据库 + 后端,数据完全离线,NSFW 自主。

所以桌面端有**两个窗口**:
1. **控制台窗口**(本文重点,需要设计):起停服务、看状态/日志、配置、检查更新、反馈。
2. **应用窗口**:加载现有的 React 游戏/创作界面(已存在,**不需要重新设计**),控制台「打开应用」时弹出。

---

## 1. 全局设计约束（硬性）

- **无 emoji、无装饰性图标字符**。状态、强调一律用**文字 + 边框 + 高亮 + 色点**表达(项目铁律)。如需图标用线性 SVG 图标,不用 emoji。
- **深色为主**(运维工具气质),参考配色见下「色板」。也可出浅色版,但深色是默认。
- **桌面尺寸**:控制台窗口默认 760×620,最小 560×460,可缩放。**不是移动端**,但要响应窗口缩放(窄到 560 时纵向堆叠)。
- **中文文案,零 AI 味**(不用「为您」「贴心」「轻松」这类)。文案表见 §11。
- **信息密度适中**:运维面板,信息要一眼可读,但不堆砌。
- **状态语义色**:运行=绿、忙(启动/停止中)=蓝(脉冲)、停止=灰、错误=红。色点而非 emoji。

## 2. 信息架构(控制台窗口)

单窗口、单列纵向滚动,自上而下分区(卡片式):

```
┌─ 顶栏 ────────────────────────────────────┐
│ Stellatrix  v0.5.0            ● 运行中     │   品牌 + 版本 + 状态指示灯(常驻)
├─ 运行模式 ────────────────────────────────┤
│ [ 在线(连云端) | 本地部署 ]   模式说明文字  │   分段切换
├─ 主操作 ──────────────────────────────────┤
│ [ 打开应用 ]  [ 在浏览器打开 ]              │
│ ── 本地模式时多出:────────────────         │
│ [启动服务] [停止] [重启]                    │
│ 状态:就绪  后端端口:51823  DB端口:15432   │
├─ 日志 ────────────────────────────[清屏][目录]┤
│ 12:00:01 [pg] pg_ctl start :15432         │   实时滚动日志面板
│ 12:00:03 [migrate] 应用迁移完成            │
│ 12:00:05 [backend] uvicorn 就绪            │
├─ 配置(可折叠)────────────────────────────┤
│ 云端地址 / 后端端口 / 更新渠道 / 自动启动   │
│ 额外环境变量 / [保存] [打开数据目录]        │
├─ 更新 ────────────────────────────────────┤
│ [检查更新]  已是最新 / 发现 vX.Y.Z [下载]   │
├─ 反馈(P1)────────────────────────────────┤
│ 问题描述 + 邮箱(收结果) + [提交]           │
└───────────────────────────────────────────┘
```

设计师可以重排/分页/加侧边导航,但以上**每个区块的功能与状态都要覆盖**。

## 3. 顶栏

- 左:产品名 `Stellatrix` + 版本号 `v0.5.0`(次级色,小字)。
- 右:**状态指示灯 chip**(常驻,任何屏都可见):色点 + 文字。五种状态:

| 状态 | 色点 | 文字 | 动画 |
|---|---|---|---|
| stopped | 灰 | 已停止 | 无 |
| starting | 蓝 | 启动中 | 脉冲 |
| running | 绿 | 运行中 | 无 |
| stopping | 蓝 | 停止中 | 脉冲 |
| error | 红 | 错误 | 无 |

- 顶栏区域可作为窗口拖拽区(macOS 无边框窗口时尤其)。

## 4. 运行模式切换

- 分段控件(两段):**在线(连云端)** / **本地部署**。当前选中高亮(主强调色填充)。
- 切换下方有一行说明:
  - 在线:「连接云端服务器,即开即用,数据存于你的云端账号。」
  - 本地:「在本机启动数据库 + 后端,数据完全离线,NSFW 自主。首次启动需初始化,稍候。」
- 切换模式应即时改变下方「主操作区」的内容(本地模式多出服务控制)。

---

## 5. 主操作区

- **主按钮「打开应用」**(最大、主强调色):
  - 在线模式 → 直接打开应用窗口加载云端。
  - 本地模式 → 若服务未起,先起(显示进度)再打开。
- 次按钮「在浏览器打开」(幽灵按钮):在系统默认浏览器打开,而非内嵌窗口。
- **本地模式专属**(在线模式隐藏):
  - 三个按钮:**启动服务 / 停止 / 重启**。按状态禁用(运行中禁用「启动」,停止时禁用「停止/重启」,忙时全禁用)。
  - 三个只读字段:**状态**(详情文字,如"就绪 http://127.0.0.1:51823"/"应用数据库迁移…")、**后端端口**、**数据库端口**。

---

## 6. 日志面板

- 等宽字体、深色背景、可滚动(高度约 180px,可设计成可拉高)。
- 每行:`时:分:秒 [来源] 内容`。来源标签次级色:`pg` / `migrate` / `backend` / `supervisor` / `更新`。
- 右上角操作:**清屏**、**打开日志目录**(打开文件管理器到日志文件夹)。
- 自动滚到底,但用户手动上滚时不抢滚动(回到底部才恢复自动)。
- 错误行可考虑红色着色。

---

## 7. 配置区(可折叠)

折叠面板(默认收起)。字段:
- **云端地址**(文本框,在线模式用,默认 `https://play.stellatrix.icu`)。
- **后端端口**(数字,0 = 自动选空闲)。
- **更新渠道**(下拉:stable / beta)。
- **自动启动**(开关:本地模式「打开应用」时自动起服务)。
- **额外环境变量**(多行文本,每行 `KEY=VALUE`,高级,默认折叠/弱化)。
- 底部:**保存配置** 按钮 + **打开数据目录** 按钮。
- 说明文字:「LLM 的 API Key 在应用内『设置』里填(BYOK),不在此处。数据(存档/数据库)存于本机,卸载默认保留。」

---

## 8. 更新区

- **检查更新** 按钮 + 一行状态文字。状态演进:
  - 检查中… → 已是最新版本 / 发现新版本 vX.Y.Z
  - 发现新版本 → 出现 **下载更新** 按钮
  - 下载中 → **进度条**(百分比)
  - 下载完成 → **重启并安装** 按钮(主强调色)
  - 出错 → 红字错误说明
- 进度条:细条(6px),主强调色填充。

---

## 9. 反馈区(P1,本轮新增)

- 标题「反馈问题」。
- **问题描述**(多行文本,必填)。
- **邮箱**(单行,选填,提示:「留邮箱,我们处理后把结果发给你」)。
  - 重要交互说明:若用户填的邮箱与其登录邮箱一致,后台会自动并入同一账户的反馈历史(设计上无需特殊处理,但可在邮箱框下加一行浅色提示:「若与你的登录邮箱一致,将并入你的账户反馈记录」)。
- **提交** 按钮。提交后:成功态(「已收到,谢谢」)/ 失败态(可重试)。
- 可附「自动附带版本与运行环境」勾选(默认勾选,帮助排查),展示为浅色说明而非显眼控件。

## 10. 首次运行向导(onboarding,需设计)

首次打开(无 config)时,先走一个 2–3 步向导而非直接进控制台:

1. **欢迎 + 选模式**:简述两种模式,让用户选「在线」或「本地」。多数人选在线。
2. **(选了本地时)初始化**:说明「将下载/初始化本地数据库,首次约需 1–2 分钟」,带进度(initdb→迁移→就绪),可看实时日志。
3. **完成**:进入控制台,高亮「打开应用」。

向导要能跳过/返回。设计上是全屏卡片步骤,与控制台同一视觉语言。

## 13. 安全卸载流程(界面)

卸载不是简单删除。设计一个流程(可在「配置」里有「卸载/重置」入口,或交系统卸载器):
1. 先**停止服务**(确保无孤儿进程/锁)。
2. 询问:**保留 / 删除 用户数据**(存档、数据库)。默认保留。选「删除」需**二次确认**,并提示「建议先备份」。
3. 清理开机自启 / 注册表项。
4. 移除应用。

## 14. 平台差异

- **macOS**:无边框/统一标题栏风格更原生;红绿灯交通灯在左;顶栏可做拖拽区。DMG 安装。
- **Windows**:标准标题栏;NSIS 安装器(可选安装目录);SmartScreen 首次可能告警(若未购代码签名证书)。
- 控制台 UI 两平台共用一套(Web 技术渲染),仅窗口 chrome 不同。
