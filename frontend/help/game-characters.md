# 角色面板

## 这页能干嘛

角色面板（标签栏上显示为"人物"）展示当前游戏里所有与玩家产生关联的角色信息：谁在当前场景里、你和他们的关系是什么、哪些角色被链接到你的角色卡库。你还可以在这里手动记录或修改与某个 NPC 的关系状态，或者把某个角色名插入输入框快速 @mention。

![角色面板](./screenshots/game-characters-panel.png)

> 注意：这里只是运行时的角色索引，**不是**角色卡的创建/编辑入口。想把某个 NPC 提升为长期角色卡，需要到平台的「角色卡」页操作。

---

## 关键概念

### 三层角色数据

面板里的角色信息分三个区域，分别来自不同数据源：

**1. 当前在场**
当前房间或战斗里实际出现的实体列表。数据来自后端在进入房间（`enter_room`）或开始战斗（`start_encounter`）时同步的运行时索引，以及战斗中的 combatants 列表作为兜底。每个在场角色会显示类型标签（NPC / 敌人 / 盟友）和当前 HP（战斗模式下）。

如果当前没有进入模组房间，这里会显示"暂无在场人物"——这是正常状态，角色进入场景后会自动出现。

**2. 关系**
玩家与特定角色之间的明确态度记录。状态值可以是"信任""警惕""敌意"或任何自定义文字。

关系有两种来源：
- **GM 自动写入**：GM 叙事输出包含结构化的关系标签（形如"关系: 玩家 → 顾承砚"）时，系统自动提取并写入
- **玩家手动添加**：点击关系区域底部的 **+ 添加关系** 按钮

关系状态的颜色标记：绿色 = 友好/信任，橙色 = 警惕，蓝色 = 亲近，红色 = 敌意。

**3. 已固定角色卡**
当前在场实体里，链接了平台角色卡（`card_id` 不为空）的角色。这些角色在平台的角色卡库里有完整的设定文件。

### 角色卡与卡牌系统的关系

游戏内的角色面板是**只读索引**，不创建或导出角色卡。卡牌系统（`user_cards`）是平台层的长期资产。如果你想把某个 NPC 固化为可复用的角色卡，需要前往平台首页的「角色卡」页操作，那里有"从 NPC 提升"的功能入口。

## 常见任务

**查看当前场景里有谁**
直接看"当前在场"区域。每张卡片显示角色名、类型（NPC/敌人/盟友）、当前 HP（战斗中）和简短的角色说明。

![当前在场角色列表](./screenshots/game-characters-active.png)

**手动添加一段关系记录**
点击"关系"区域底部的 **+ 添加关系** → 弹窗里输入 NPC 名字 → 输入关系状态（如"信任"）→ 确认。

**修改与某角色的关系状态**
找到该角色的关系卡片 → 点击绿色/橙色/红色的状态标签 → 直接输入新状态 → 回车保存。

**删除一条关系记录**
找到该角色的关系卡片 → 点击右上角的 **×** → 确认删除弹窗。

**把角色名插入输入框**
在任意角色卡片右上角，点击 **@** 图标。如果浏览器支持，角色名会直接插入到游戏输入框；否则会复制到剪贴板，手动粘贴即可。

也可以直接把角色卡片**拖拽**到输入框区域，会自动插入 `@角色名`。

![拖拽角色卡到输入框](./screenshots/game-characters-drag-mention.png)

## 常见问题

**某个 NPC 在剧情里出现了，但角色面板里没有显示，为什么？**
"当前在场"列表只显示后端已同步的运行时实体——这些实体在进入房间或触发合法战斗时由系统写入。如果 NPC 只是在 GM 的叙事文本里出现，但没有走过 `enter_room` 或 `start_encounter` 流程，就不会自动出现在这里。可以手动在"关系"区域为该角色添加一条记录，作为临时追踪方式。

**关系状态对 GM 有影响吗？**
有。GM 在生成回复时会参考 `state.relationships` 数据，关系状态为"敌意"的角色，GM 在叙事里会倾向于让其表现敌对行为。

**为什么没有"创建角色卡"按钮？**
这是设计决策：游戏内只显示运行时实体，不在游戏中创建持久化角色卡，防止误操作和数据碎片化。长期角色卡的创建、编辑、提升都在平台层的「角色卡」页完成。

**"已固定角色卡"区域是空的，正常吗？**
正常。只有当前在场实体里存在 `card_id` 链接时，这个区域才会显示。如果你没有在平台角色卡页把某个 NPC 和存档里的实体关联起来，这里就会是空的。
